**4. 핵심 기술 구현 사항**
   
   * 4-1. 객체 지향적 타격 분리: 평타는 플레이어 애니메이션 이벤트에 종속된 형태로, 스킬은 독립된 네트워크 투사체로 완전히 분리되어 작동합니다.
   
   * 4-2. 컴포지션 설계: 무기 -> 스킬 -> 샤드가 런타임에 결합하여 최종 데미지와 범위를 결정합니다.

   * 4-3. 네트워크 동기화 철학: 물리 이동과 핵심 로직(데미지 판정, 스킬 생성, 글로벌 이벤트 대기)등은 100% 서버 권한(!IsServer 및 RPC)으로 처리하고,
   클라이언트는 이펙트 재생 및 입력 전달만 담당하는 Authoritative Server(서버 권위) 구조를 따르고 있습니다.


**5. 트러블 슈팅 히스토리**

1) NetworkManager와 수동 스폰 시스템의 충돌로 인한 중복 스폰 이슈
   
* 문제 상황 : 멀티플레이 룸 진입 및 인게임 씬 전환 시, 각 클라이언트마다 플레이어 캐릭터가 항상 2개씩 중복 생성되는 현상 확인.
* 원인 분석 : Unity Netcode for GameObjects의 NetworkManager 인스펙터 내 'Player Prefab' 슬롯에 등록된 자동 스폰 시스템과, 게임 진입 처리를 담당하는 전역 시스템 NetworkTransmission에서 호스트/클라이언트 연결 시 NetworkObject.SpawnAsPlayerObject()를 호출하는 수동 RPC 스폰 로직이 동시에 트리거되면서 객체가 중복 스폰됨.
* 해결 방안 : NetworkManager 내부의 기본 Player Prefab 등록을 해제하여 엔진 레벨의 자동 스폰을 차단함. 이후 NetworkTransmission 매니저를 통해서만 세션 연결 상태 및 클라이언트 ID를 검증한 후, 서버 권한으로 명시적 수동 스폰을 수행하도록 동기화 파이프라인을 일원화하여 해결함.

2) 멀티플레이어 환경에서의 네트워크 물리 동기화 프레임 드롭(지터링) 이슈
   
* 문제 상황 : 로컬 클라이언트 환경에서는 플레이어의 이동이 매끄럽게 연출되나, 원격 클라이언트 시점에서 상대방 캐릭터의 이동을 관측할 때 화면이 뚝뚝 끊기거나 순간 이동하는 지터링 및 위치 오차 현상 발생.
* 원인 분석 : 서버 권한 이동 동기화 과정에서, 원격 캐릭터가 물리 연산 주기가 아닌 네트워크 패킷 수신 주기에 맞춰서 위치를 갱신하기 때문에 발생하는 현상으로 사료됨. 또한 Rigidbody2D 컴포넌트의 물리 보간 기능이 활성화되지 않아 패킷 사이의 공백 프레임이 부드럽게 메워지지 않을 수 있겠다는 생각.
* 해결 방안 : 원격 클라이언트가 조작하는 모든 플레이어 프리팹의 Rigidbody2D 내 보간 옵션을 Interpolate로 변경하여 직전 프레임과 현재 프레임 사이의 물리 위치를 부드럽게 선형 보간함.
  수신된 데이터와 물리 루프 간의 타이밍을 정밀하게 일치시켜 프레임 드롭 현상을 해결, 대역폭 제한 환경에서도 부드러운 이동 동기화를 보장함.

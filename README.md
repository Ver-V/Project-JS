**4. 핵심 기술 구현 사항**
   
   * 4-1. 객체 지향적 타격 분리: 평타는 플레이어 애니메이션 이벤트에 종속된 형태로, 스킬은 독립된 네트워크 투사체로 완전히 분리되어 작동합니다.
   
   * 4-2. 컴포지션 설계: 무기 -> 스킬 -> 샤드가 런타임에 결합하여 최종 데미지와 범위를 결정합니다.
    
   * 4-3. 네트워크 동기화 철학: 물리 이동과 핵심 로직(데미지 판정, 스킬 생성, 글로벌 이벤트 대기)은 100% 서버 권한(!IsServer 및 RPC)으로 처리하고,
   클라이언트는 이펙트 재생 및 입력 전달만 담당하는 Authoritative Server(서버 권위) 구조를 따르고 있습니다.


**5. 트러블 슈팅 히스토리**

   * 5-1 : 유저가 게임을 시작하면 2명이 스폰되는 문제.
     문제 상황 : 로비에 들어갈 때까지만 해도 하나만 스폰하는 플레이어가, 인게임에 들어가면 항상 두 명씩 생기기 시작했다.
     원인 분석 : 아마도 프리팹을 둘 다 넣었거나, 아니면 어딘가에서 두 번 스폰하고 있을 것이라고 생각했다.
     문제 해결 : NetworkTransmission과 NetworkObject에 플레이어 프리팹을 둘 다 넣어놔서 생기던 문제였기에, NetworkObject에서 프리팹을 빼니 간단하게 해결되었다.
   
   * 5-2 : Rigidbody 에서의 보간 처리 및 NetworkRigidbody
     문제 상황 : 내 컴퓨터에서 볼 때는 내 플레이어 캐릭터가 정상적으로 움직이는데, 남의 네트워크에서 내 캐릭터를 확인하면 뭔가 뚝뚝 끊기면서 제대로 움직이지 않았다.
     원인 분석 : 아마도 네트워크의 품질에 문제가 있는 것 같다고 생각해 여러가지 처리를 해보았지만, 역시나 큰 효과가 있진 않았다.
     문제 해결 : 같은 프로젝트를 하는 팀원이 Interpolation의 문제일 가능성이 높다고 얘기하며 체크하고 테스트를 해보라고 말해주어, 체크하고 테스트하니 확실히 움직임이 부드러워지며 문제가 해결되었다.
